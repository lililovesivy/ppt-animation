# PPT 动画编排核心逻辑

> **版本**：v1.1
> **用途**：为法律/知识类演示文稿设计逐元素出场动画的通用编排规则
> **来源**：基于「血金」案例拆解 PPT 的多次迭代经验总结

---

## 一、核心原则

| 原则 | 说明 |
|------|------|
| **逐元素出场** | 每个独立元素单独动画，拒绝整页淡入 |
| **读序即出场序** | 元素按阅读顺序（上→下、左→右）依次出现 |
| **容器先于内容** | 框/卡片/背景色块先出现，内部文字/图片后出现 |
| **动静分层** | 标题有动感，正文平稳，重点有强调 |
| **节奏明快** | 每个动画 0.3–0.5 秒，不拖沓 |

---

## 二、出场顺序规则

### 同一页面内的标准出场序列

```
1. 页面框架（背景/顶部条）     → 无动画，始终可见
2. 页面标签/副标签（如"案例拆解"）→ 淡入
3. 主标题                     → 逐字飞入，从左到右
4. 关键强调词/核心概念         → 飞入/缩放 → 自动脉冲强调（0.1s 后）
5. 正文/说明文字               → 缩放/浮入
6. 信息卡片背景                 → 浮入
7. 卡片内标签（如"概念提示"）    → 淡入
8. 卡片内文字                   → 淡入/浮入
9. 卡片内配图/图标              → 缩放（多图依次）
10. 底部文字/脚注               → 浮入
11. （可选）退场                 → 内容切换时，旧元素淡出/飞出
```

### 同层级元素排序

- 从上到下，从左到右（符合中文阅读习惯）
- 重要 → 次要
- 结构性元素（框、线）先于内容性元素（文字、图）

## 三、元素动画映射规则（完整版）

每个元素的生命周期动画由三段构成：**入场 → 强调 → 退场**。三段不是都需要的，根据元素角色判定。

### 3.1 映射总表

| 元素角色 | 入场动画 | 强调动画 | 退场动画 | 判断依据 |
|---------|---------|---------|---------|---------|
| **页面大背景** | — | — | — | 始终可见，不设任何动画 |
| **主标题** | 逐字飞入（左→右）| — | — | 标题本身是页面锚点，入场即够 |
| **副标题/页面标签** | 淡入 | — | — | 信息性元素，点到为止 |
| **核心关键词** | 缩放 | **脉冲缩放** | — | 入场后 0.1s 自动触发强调，加深印象 |
| **重要数字/数据** | 缩放 | **脉冲缩放** | — | 数字有冲击力，强调后停留 |
| **正文/说明文字** | 缩放 / 浮入 | — | 淡出（可选） | 多批内容逐段替换时，前一批需退场 |
| **注意/警示文字** | 旋转进入 | — | — | 旋转有"翻转注意"的隐喻 |
| **高亮色块** | 浮入 | — | — | 纯装饰，不需要强调 |
| **卡片/信息框背景** | 浮入 | — | 飞出（可选） | 卡片作为整体切换时，整框飞出 |
| **卡片标签** | 淡入 | — | — | 与卡片背景同步或紧随其后 |
| **卡片内容文字** | 浮入 / 淡入 | — | 淡出（可选） | 与正文同理 |
| **卡片配图/图标** | 缩放 | — | 飞出（可选） | 图片缩放入场有冲击力；多图依次入场 |
| **底部文字/脚注** | 浮入 | — | — | 收尾信息，不需要二次强调 |
| **装饰形状/线条** | 无动画 或 淡入 | — | — | 只起装饰作用，可忽略或低调入场 |

### 3.2 强调动画规则

强调动画是在元素**入场完成后**自动追加的二次动作，用于突出关键信息。

#### 什么时候需要强调？

| 场景 | 强调类型 | 时机 | 示例 |
|------|---------|------|------|
| 核心概念词 | 脉冲缩放 | 入场后立即（0.1s） | "能。" — 一个字概括全页 |
| 关键数字/比例 | 脉冲缩放 | 入场后立即（0.1s） | "+1/3"、"120万元" |
| 结论性短句 | 脉冲缩放 | 入场后立即（0.1s） | "决定权 在家属手里" |
| 警示/禁止标记 | 脉冲缩放 | 入场后立即（0.1s） | "⚠️ 注意" |

#### 什么时候不需要强调？

- 主标题（字号已足够大，飞入已有关注度）
- 正文段落（长文字，强调会分散阅读注意力）
- 卡片背景/色块（结构性元素，喧宾夺主）
- 普通图片/图标（非关键信息）

#### 强调动画的执行方式

```
入场动画（click触发）→ 停留0.1s → 强调动画（自动执行，与上一动画同时）
```

- 强调动画设为「与上一动画同时」模式
- 不额外占用点击次数，在入场后自动追加

### 3.3 退场动画规则

退场用于**同一页内内容切换**或**元素不再需要展示时**。

| 场景 | 退场动画 | 示例 |
|------|---------|------|
| 多批正文逐段替换 | 前一段 → 淡出，后一段 → 入场 | 论据 A 说完后退场，论据 B 入场 |
| 卡片整体切换 | 整框 → 飞出 | P3 右侧框 + 配图依次飞出 |
| 过渡到下一核心结论 | 旧结论文字 → 淡出 | 清除旧信息，聚焦新观点 |
| 静态展示页 | **不退场** | 绝大多数情况不需要退场 |

#### 退场 ≠ 翻页

翻页时所有元素自动消失，不需要额外设退场动画。退场只用于**同一页内的元素更替**。

### 3.4 不设动画的元素

- **页面大背景**：始终可见，作为视觉锚点
- **装饰性复杂底纹**：避免干扰阅读层次
- **纯分割线/边框**：可设淡入，可不设

## 四、容器优先原则（Container-First Rule）

当一个页面上存在多个**视觉区块**（如左文右图的卡片、顶部导航条、底部信息栏），必须遵循：

```
容器背景 → 容器标题/标签 → 容器内容文字 → 容器内图片/图标
```

### 示例：右侧「概念提示」卡片

入场顺序：
1. 灰色卡片背景（浮入）
2. "概念提示" 标签（淡入）
3. 黄色文字底色条（浮入）
4. 说明文字（淡入）
5. 三张配图（依次缩放）

**禁止**：配图在卡片背景之前出现，或文字在标签之前出现。

## 五、停留时间与节奏

### 动画时长

| 动画类型 | 时长 | 适用场景 |
|---------|------|---------|
| 飞入 | 0.3–0.5s | 标题、强调词 |
| 淡入 | 0.3–0.4s | 标签、小字 |
| 浮入 | 0.4–0.5s | 卡片、容器 |
| 缩放 | 0.4–0.5s | 图片、正文 |
| 脉冲 | 0.3s | 强调词二次动作 |

### 触发方式

- **全部点击触发**：讲者控制节奏，每点一次出一个元素
- 特殊场景可用「与上一动画同时」组合容器和标签

### 页间过渡

- 默认保留简单的切页效果即可
- 如需强调切换，使用淡入淡出切页（0.3s）

## 六、典型页面模板

### 模板 A：标题 + 正文 + 配图

```
1. 顶部标签              → 淡入
2. 主标题（大字）          → 逐字飞入
3. 正文说明              → 缩放
4. 右侧配图              → 缩放
```

### 模板 B：标题 + 信息卡片 + 图示

```
1. 顶部标签              → 淡入
2. 主标题                → 飞入
3. 卡片背景              → 浮入
4. 卡片标签              → 淡入
5. 卡片文字              → 浮入
6. 卡片配图              → 缩放（多图则依次缩放）
7. 底部说明              → 浮入
```

### 模板 C：单个核心概念强调

```
1. 页面标签              → 淡入
2. 核心关键词（超大字号）   → 缩放入场 → 自动脉冲强调
3. 解释文字              → 缩放
```

### 模板 D：同页内容切换（需要退场）

```
1. 顶部标签              → 淡入
2. 主标题                → 飞入
3. 第一批正文            → 浮入
4. 第一批正文            → 淡出（退场）
5. 第二批正文            → 浮入（入场）
6. 配图                  → 缩放
```

## 七、检查清单

添加动画后逐项自查：

- [ ] 是否有元素在页面加载时「裸奔」出现（没有出场动画）？
- [ ] 容器背景是否比容器内容先出现？
- [ ] 配图是否出现在所属文字之后（或同步）？
- [ ] 核心关键词/数据是否追加了强调动画（脉冲）？
- [ ] 强调动画是否设为自动执行（不额外占用一次点击）？
- [ ] 是否有多种动画混用，层次清晰？
- [ ] 节奏是否均匀，没有某一步特别慢或特别快？
- [ ] 退出动画是否必要？（同一页内容切换才需要，翻页不需要）

## 八、常见错误及修正

| 问题 | 原因 | 修正 |
|------|------|------|
| 页面加载时某元素已可见 | 该元素未设出场动画，或使用了退出动画而非入场动画 | 为该元素添加入场动画，并确保是触发式 |
| 配图比文字先出现 | 出场顺序排列错误 | 将配图的动画移到对应文字之后 |
| 核心词入场后没有强调 | 遗漏了强调动画 | 追加脉冲缩放强调，设「与上一动画同时」|
| 强调动画占用额外点击 | 强调动画设为了点击触发 | 改为自动执行（与上一动画同时） |
| 效果太花哨 | 同一页使用了超过 3 种不同动画类型 | 精简为 2–3 种，保持克制 |
| 某一步太久没反应 | 动画设置了过长的延迟 | 去掉延迟，或缩短至 0.1s 以内 |
| 退场后页面留白 | 仅设了退场，未跟进新内容入场 | 退场后紧跟新内容的入场动画 |

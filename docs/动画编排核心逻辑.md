# PPT 动画编排核心逻辑

> **版本**：v1.0
> **用途**：为法律/知识类演示文稿设计逐元素出场动画的通用编排规则
> **来源**：基于「血金」案例拆解 PPT 的多次迭代经验总结

---

## 一、核心原则

| 原则 | 说明 |
|------|------|
| **逐元素出场** | 每个独立元素单独动画，拒绝整页淡入 |
| **读序即出场序** | 元素按阅读顺序（上→下、左→右）依次出现 |
| **容器先于内容** | 框/卡片/背景色块先出现，内部文字/图片后出现 |
| **动静分层** | 标题有动感，正文平稳，重点有强调 |
| **节奏明快** | 每个动画 0.3–0.5 秒，不拖沓 |

---

## 二、出场顺序规则

### 同一页面内的标准出场序列

```
1. 页面框架（背景/顶部条）     → 无动画，始终可见
2. 页面标签/副标签（如"案例拆解"）→ 淡入
3. 主标题                     → 逐字飞入，从左到右
4. 关键强调词/核心概念         → 飞入 → 脉冲强调
5. 正文/说明文字               → 缩放/浮入
6. 信息卡片背景                 → 浮入
7. 卡片内标签（如"概念提示"）    → 淡入
8. 卡片内文字                   → 淡入/浮入
9. 卡片内配图/图标              → 缩放
10. 底部文字/脚注               → 浮入
```

### 同层级元素排序

- 从上到下，从左到右（符合中文阅读习惯）
- 重要 → 次要
- 结构性元素（框、线）先于内容性元素（文字、图）

---

## 三、元素类型 → 动画类型映射

### 按元素角色匹配

| 元素角色 | 推荐动画 | 说明 |
|---------|---------|------|
| **主标题** | 飞入（左→右）| 有动感，引导视线从左到右扫描 |
| **副标题/页面标签** | 淡入 | 低调入场，不抢主标题注意力 |
| **关键词强调** | 飞入 + 脉冲 | 先入场，再震动强调，加深印象 |
| **正文/说明文字** | 缩放 / 浮入 | 自然展开，阅读友好 |
| **卡片/信息框背景** | 浮入 | 先立框架，再填内容 |
| **卡片内标签** | 淡入 | 小字用淡入，不突兀 |
| **卡片内配图** | 缩放 | 有视觉冲击力，适合示意图片 |
| **底部文字** | 浮入 | 收尾感，自然过渡到下一内容 |

### 不设动画的元素

- **页面大背景**：始终可见，作为视觉锚点
- **装饰性复杂底纹**：避免干扰阅读层次

---

## 四、容器优先原则（Container-First Rule）

当一个页面上存在多个**视觉区块**（如左文右图的卡片、顶部导航条、底部信息栏），必须遵循：

```
容器背景 → 容器标题/标签 → 容器内容文字 → 容器内图片/图标
```

### 示例：右侧「概念提示」卡片

入场顺序：
1. 灰色卡片背景（浮入）
2. "概念提示" 标签（淡入）
3. 黄色文字底色条（浮入）
4. 说明文字（淡入）
5. 三张配图（依次缩放）

**禁止**：配图在卡片背景之前出现，或文字在标签之前出现。

---

## 五、停留时间与节奏

### 动画时长

| 动画类型 | 时长 | 适用场景 |
|---------|------|---------|
| 飞入 | 0.3–0.5s | 标题、强调词 |
| 淡入 | 0.3–0.4s | 标签、小字 |
| 浮入 | 0.4–0.5s | 卡片、容器 |
| 缩放 | 0.4–0.5s | 图片、正文 |
| 脉冲 | 0.3s | 强调词二次动作 |

### 触发方式

- **全部点击触发**：讲者控制节奏，每点一次出一个元素
- 特殊场景可用「与上一动画同时」组合容器和标签

### 页间过渡

- 默认保留简单的切页效果即可
- 如需强调切换，使用淡入淡出切页（0.3s）

## 六、典型页面模板

### 模板 A：标题 + 正文 + 配图

```
1. 顶部标签              → 淡入
2. 主标题（大字）          → 逐字飞入
3. 正文说明              → 缩放
4. 右侧配图              → 缩放
```

### 模板 B：标题 + 信息卡片 + 图示

```
1. 顶部标签              → 淡入
2. 主标题                → 飞入
3. 卡片背景              → 浮入
4. 卡片标签              → 淡入
5. 卡片文字              → 浮入
6. 卡片配图              → 缩放（多图则依次缩放）
7. 底部说明              → 浮入
```

### 模板 C：单个核心概念强调

```
1. 页面标签              → 淡入
2. 核心关键词（超大字号）   → 飞入 → 脉冲
3. 解释文字              → 缩放
```

## 七、检查清单

添加动画后逐项自查：

- [ ] 是否有元素在页面加载时「裸奔」出现（没有出场动画）？
- [ ] 容器背景是否比容器内容先出现？
- [ ] 配图是否出现在所属文字之后（或同步）？
- [ ] 是否有多种动画混用，层次清晰？
- [ ] 节奏是否均匀，没有某一步特别慢或特别快？
- [ ] 退出动画是否必要？（非必要不要设退出动画）

---

## 八、常见错误及修正

| 问题 | 原因 | 修正 |
|------|------|------|
| 页面加载时某元素已可见 | 该元素未设出场动画，或使用了退出动画而非入场动画 | 为该元素添加入场动画，并确保是触发式 |
| 配图比文字先出现 | 出场顺序排列错误 | 将配图的动画移到对应文字之后 |
| 效果太花哨 | 同一页使用了超过 3 种不同动画类型 | 精简为 2–3 种，保持克制 |
| 某一步太久没反应 | 动画设置了过长的延迟 | 去掉延迟，或缩短至 0.1s 以内 |

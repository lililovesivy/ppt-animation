# PPT动画生成工作流程

> **版本**：v1.0  
> **用途**：为法律类口播视频PPT自动添加元素级动画，替代视频动画导出，直接使用原生PPT动画  
> **适用场景**：闪婚闪离、房产纠纷、财产分割等法律主题PPT

---

## 一、核心设计原则

1. **元素级动画**：每个元素独立入场，而非整页淡入淡出
2. **类型匹配**：不同类型元素使用不同动画风格，层次分明
3. **节奏明快**：符合口播视频节奏，不拖沓，不花哨
4. **兼容性**：使用PowerPoint原生动画，确保在任何版本都能正常播放

---

## 二、元素分类与动画映射

### 已实现的固定规则（可重复使用）

| 元素类型 | 识别规则 | 动画类型 | 方向规则 | 触发方式 |
|---------|---------|---------|---------|---------|
| **背景图** | 尺寸 > 90% 页面宽高，位置接近 (0,0) | **无动画** | 固定显示，不作为出场元素 | 无 |
| **标题文字** | 短文本（<30字），字号 > 24pt 或位于页面上部 | 擦除 | 自左向右 | 点击触发 |
| **正文文字** | 长文本，字号中等 | 擦除 | 自左向右 | 点击触发 |
| **重点文字** | 红色文字、加粗强调 | 飞入 | 从下方弹入 | 点击触发 |
| **内容图片** | 非背景的图片（截图、示意图、人物图等） | 飞入 | 从最近的边缘/角落飞入 | 点击触发 |
| **小图标** | 宽度 < 200px 的小图片 | 飞入 | 从最近的边缘飞入 | 点击触发 |
| **装饰形状** | 无文字的几何形状，用于装饰 | 飞入 | 从最近的边缘飞入 | 点击触发 |
| **高亮框** | 宽高比 > 2 的扁平形状，用于突出文字 | 擦除 | 自左向右 | 点击触发 |
| **线条** | 细长形状（宽高比 > 8）或线条类型 | 擦除 | 从左到右 / 从上到下 | 点击触发 |

## 三、动画方向智能匹配算法

### 核心逻辑：从靠近的边缘飞入，营造自然感

1. **计算元素中心点**相对于页面的位置 (rel_x, rel_y)
2. **计算到四边的距离**：左、右、上、下
3. **选择最近的边**作为飞入方向
4. **增强效果**：对于图片和装饰形状，如果靠近角落，使用**对角线方向**（如左上、右下），增加动感

### 8个飞入方向
```
左上(left-top)    上(top)    右上(right-top)
    \                |                /
     \               |               /
左(left) ————————————●———————————— 右(right)
     /               |               \
    /                |                \
左下(left-bottom)  下(bottom)  右下(right-bottom)
```

### 4个擦除方向
```
上(top) → 从上往下
下(bottom) → 从下往上
左(left) → 从左往右
右(right) → 从右往左
```

## 四、动画出场顺序编排

### 排序优先级（从先到后）

1. **标题文字** → 明确主题
2. **高亮框** → 提前铺垫强调区域
3. **正文文字** → 展开内容
4. **内容图片** → 视觉佐证
5. **装饰形状/小图标** → 细节点缀
6. **重点强调元素** → 最后突出，加深印象

### 同层级排序
- 从上到下，从左到右（符合阅读习惯）
- 重要内容优先，次要内容靠后

## 五、技术实现方案

### 动画原理
基于PowerPoint原生动画XML结构，直接修改PPT内部XML文件实现。

### 两种基础动画模板

#### 1. 飞入动画 (Fly In)
- **presetID**: 2
- **实现方式**: 位移动画 (p:anim)
- **支持方向**: 8个（含对角线）
- **适用元素**: 图片、形状、图标
- **特点**: 有动感，适合强调元素

#### 2. 擦除动画 (Wipe)
- **presetID**: 22
- **实现方式**: 滤镜动画 (p:animEffect)
- **支持方向**: 4个（上/下/左/右）
- **适用元素**: 文字、高亮框、线条
- **特点**: 平滑自然，适合文字内容

### 文本构建方式
- **整体出现** (all_at_once): 整个文本框一次性出现
- **按段落出现** (by_paragraph): 逐段依次出现（多用于正文）
- 通过 `p:bldLst` 和 `grpId` 控制

## 六、使用方式

### 命令行调用
```bash
python3 gen_anim_v5.py
```

### 输入输出
- **输入**: 原始PPT文件（无动画或有基础动画）
- **输出**: 带动画的PPT文件（.pptx格式，原生兼容）

### 配置说明
在脚本头部 `ELEMENT_ANIMATION_CONFIG` 字典中可自定义：
- 每种元素的动画类型
- 默认方向
- 触发方式
- 文本构建方式

## 九、注意事项

1. **兼容性优先**: 只使用PowerPoint标准动画，确保跨版本兼容
2. **内容不变**: 只添加动画，不修改任何文字、图片、布局
3. **适度原则**: 动画服务于内容表达，不过度炫技
4. **节奏适配**: 法律类口播节奏偏快，动画时长控制在0.5-1秒
5. **可编辑性**: 生成的PPT保持完全可编辑，方便后续调整
